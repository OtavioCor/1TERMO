# 🎓 Repositório de Planos de Aula e Materiais Didáticos

Bem-vindo ao repositório central de conteúdos programáticos e guias de aula. Este espaço reúne o planejamento pedagógico e os materiais técnicos estruturados para capacitar estudantes em arquitetura de hardware, desenvolvimento de software multiplataforma e engenharia de sistemas.

---

## 📚 Disciplinas e Conteúdos Programáticos

### 1. 🤖 Arquitetura IoT (Internet das Coisas)
Focada na integração entre o mundo físico e o digital, abordando desde a captura de dados na borda até o armazenamento em nuvem.
*   **Tecnologias**: Ecossistema Arduino (UNO, ESP32), C++ (Firmware), Python (Gateways).
*   **Principais Conceitos**: Camadas de percepção e rede, eletrônica digital/analógica, barramentos de comunicação, protocolos MQTT/HTTP e segurança em IoT.

### 2. 📋 Engenharia e Levantamento de Requisitos
Dedicada às etapas críticas de descoberta, especificação e validação das reais necessidades de um produto de software.
*   **Metodologias & Técnicas**: Brainstorming, entrevistas estruturadas e engenharia de valor com Stakeholders.
*   **Modelagem & Entrega**: Requisitos Funcionais (RF) e Não Funcionais (RNF), diagramas UML (Casos de Uso, Atividades, DFD), prototipagem de baixa/alta fidelidade (Figma) e relatórios técnicos (SRS).

### 3. 💻 Lógica de Programação Multiplataforma
Base fundamental de raciocínio algorítmico e estruturação de código adaptada para operação nos principais sistemas operacionais do mercado.
*   **Fundamentos Lógicos**: Variáveis, tipos de dados primitivos, estruturas condicionais (`if/else`), laços de repetição (`while/for`) e modularização (funções).
*   **Ambientes Operacionais**:
    *   🪟 **Windows**: Prompt de Comando (CMD), PowerShell e variáveis de ambiente.
    *   🐧 **Linux**: Terminal Bash, manipulação de arquivos via CLI e permissões.
    *   🍏 **iOS / macOS**: Linha de comando Unix (Zsh), ambientes Sandbox e ecossistema Swift Playgrounds.
*   **Versionamento**: Fluxo de trabalho essencial utilizando Git e GitHub (commits, status, push/pull).

Cada disciplina possui sua própria pasta contendo o arquivo `README.md` detalhado com exemplos de código, cronogramas de aula sugeridos e exercícios práticos. Navegue pelas pastas acima para acessar o material de cada curso.
